# 云同步数据不显示 - 诊断和修复指南

## 🔍 问题诊断

**症状：**
- 你在自己手机上添加数据
- 家人手机上看不到这些数据

**原因分析：**

### 可能的原因 1：数据没有上传到云端
- 你添加数据后，数据只保存在本地 localStorage
- 需要主动触发上传到 Supabase

### 可能的原因 2：家人手机没有拉取数据
- 家人手机配置了 Supabase，但没有自动拉取数据
- 需要手动刷新或等待自动同步

### 可能的原因 3：Supabase 表结构不匹配
- 本地数据字段名和云端表字段名不一致
- 导致数据无法正确保存或读取

---

## ✅ 修复步骤

### 第一步：检查浏览器控制台

1. **打开浏览器开发者工具**
   - 按 F12 或右键 → 检查
   - 切换到"控制台"标签

2. **查看同步日志**
   - 添加数据后，查看控制台输出
   - 应该看到类似的日志：
     ```
     ✅ 健康记录已同步到云端
     🔄 开始自动同步数据...
     ✅ 自动同步完成
     ```

3. **如果看到错误**
   - 记下错误信息
   - 可能是 Supabase 配置问题

### 第二步：验证 Supabase 配置

1. **检查 URL 和 API Key**
   - 确保你和家人使用的是**相同的** URL 和 API Key
   - 在云同步设置中查看

2. **测试连接**
   - 点击"测试连接"按钮
   - 应该显示"连接成功"

### 第三步：手动刷新数据

1. **在家人手机上**
   - 打开应用
   - 点击右上角 ☁️ 云同步按钮
   - 查看是否显示"已连接"

2. **刷新页面**
   - 按 F5 或 Ctrl+R 刷新
   - 或者关闭应用重新打开

3. **查看控制台**
   - 应该看到自动同步的日志

### 第四步：检查 Supabase 数据库

1. **登录 Supabase 控制面板**
   - 访问 https://app.supabase.com
   - 进入你的项目

2. **查看数据表**
   - 点击"SQL Editor"
   - 运行查询查看数据：
     ```sql
     SELECT * FROM health_records;
     SELECT * FROM diet_records;
     SELECT * FROM exercise_records;
     ```

3. **检查数据是否存在**
   - 如果表中有数据，说明上传成功
   - 如果表中没有数据，说明上传失败

## ❓ 常见问题

**Q: 为什么数据没有立即显示？**
A: 自动同步间隔是 30 秒，最多等待 30 秒。或者手动刷新页面。

**Q: 如何加快同步速度？**
A: 可以修改 `startAutoSync()` 中的 30000 改为更小的数字（单位毫秒）。

**Q: 如何确认数据已上传？**
A: 打开浏览器控制台，查看是否有"✅ 已同步到云端"的日志。

**Q: 如果还是看不到数据怎么办？**
A: 
1. 检查浏览器控制台是否有错误
2. 检查 Supabase 数据库中是否有数据
3. 确保两个手机使用相同的 Supabase 配置
