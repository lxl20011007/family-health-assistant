# 家庭健康助手 - 新版本部署指南

## 🚀 新版本特性

### **核心改进**
1. **用户认证系统** - 邮箱+密码登录/注册
2. **云端数据存储** - 替换 localStorage，数据永久保存

## 📋 部署步骤

### **第一步：准备 Supabase 项目**

1. **注册 Supabase**
   - 访问 https://supabase.com
   - 点击 "Start your project"
   - 使用 GitHub 或邮箱注册

2. **创建新项目**
   - 点击 "New project"
   - 输入项目名称: `family-health-assistant`
   - 设置数据库密码（记住这个密码）
   - 选择区域: `East Asia (Singapore)` 或离你最近的区域
   - 点击 "Create new project"

3. **获取项目配置**
   - 进入项目后，点击左侧菜单 "Settings" → "API"
   - 复制以下信息：
     - **Project URL**: `https://xxxxxxxx.supabase.co`
     - **anon/public key**: `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`

### **第二步：执行数据库迁移**

1. **进入 SQL Editor**
   - 在 Supabase 控制台，点击左侧菜单 "SQL Editor"
   - 点击 "New query"

2. **执行迁移脚本**
   - 打开 `supabase-migration.sql` 文件
   - 复制全部内容
   - 粘贴到 SQL Editor
   - 点击 "Run" 执行

3. **验证表创建**
   - 点击左侧菜单 "Table Editor"
   - 应该看到以下表：
     - `family_members`

     - `diet_records`

     - `medications`
     - `ai_chat_sessions`
     - `ai_chat_messages`

### **第三步：配置应用**

1. **更新 Supabase 配置**
   - 在应用中点击 "云同步设置" 按钮
   - 输入你的 Supabase URL 和 anon key
   - 点击 "测试连接" 验证配置

2. **注册用户**

   - 输入邮箱和密码（至少6位）

3. **登录使用**
   - 使用注册的邮箱密码登录
   - 系统会自动检测本地数据并提示导入

## 🔧 故障排除

### **常见问题**

#### 1. SQL 迁移失败
**错误**: `relation "family_members" does not exist`
**解决**: 先创建基础表，再执行迁移脚本

#### 2. 连接测试失败
**错误**: `Failed to fetch`
**解决**:
- 检查 URL 和 API Key 是否正确
- 检查网络连接
- 确保已执行建表脚本

#### 3. 认证失败
**错误**: `Invalid login credentials`
**解决**:
- 确认邮箱密码正确
- 检查邮箱是否已验证（可选）
- 尝试重置密码

#### 4. 实时同步不工作
**解决**:
- 检查浏览器控制台是否有错误
- 确认 Supabase Realtime 已启用
- 刷新页面重新连接

### **调试方法**

1. **浏览器开发者工具**
   - 按 F12 打开开发者工具
   - 查看 Console 标签页的错误信息
   - 查看 Network 标签页的请求响应

2. **Supabase 日志**
   - 进入 Supabase 控制台
   - 点击 "Logs" 查看 API 调用日志
   - 点击 "Auth" → "Users" 查看用户状态

3. **本地调试**
   - 使用 `console.log()` 输出调试信息
   - 检查 localStorage 中的数据
   - 验证 Supabase 连接状态

## 📱 多设备使用指南

### **同一家庭使用**
1. **所有设备使用同一账号**
   - 在所有设备上用相同的邮箱密码登录
   - 数据会自动同步

2. **数据同步流程**
   ```
   设备A添加数据 → 云端存储 → 设备B自动更新
   ```

3. **离线使用**
   - 网络断开时，数据保存在本地
   - 网络恢复后，自动同步到云端

### **不同家庭使用**
- 每个家庭注册不同的账号
- 数据完全隔离，互不可见

## 🔒 安全注意事项

### **数据安全**
- 所有数据通过 HTTPS 加密传输
- 数据库启用行级安全策略（RLS）
- 用户只能访问自己的数据

### **账号安全**
- 使用强密码（字母+数字+符号）
- 不要分享账号密码
- 定期检查登录设备

### **备份策略**
- 云端数据自动备份（Supabase 提供）
- 重要数据建议本地导出备份
- 定期检查数据完整性
